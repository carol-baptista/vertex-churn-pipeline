# Churn RF model releases

Human-readable history for the `churn-rf` serving line. Each release also
ships a machine-readable `manifest.json` inside `vN/` (generated by
`make package`).

Vertex Model Registry auto-versions each upload under `churn-predictor`
(version 1, 2, 3…). The folder name here (`v1`, `v2`) is our release tag;
keep this changelog in sync when you cut a new release.

## v1 — baseline Random Forest (production candidate)

**Released:** 2026-07-04 (local package; Registry upload pending)

**Why this version**

- First deployable release for the telco churn portfolio project.
- **Random Forest** champion: best **test** F1 (~0.620) and PR-AUC (~0.683),
  even though XGBoost wins CV F1 by a small margin.
- **Baseline features only** (18 raw EDA columns). Engineered features (+4)
  did not improve recall vs frozen baseline.

**Training defaults**

| Knob | Value |
| --- | --- |
| Feature set | `baseline` |
| Metric | `f1` (grid + selection) |
| Pos weight | `sqrt` |
| Threshold strategy | `f1` on validation |
| Split | stratified 70% / 15% / 15% |

**Headline test metrics** (Random Forest @ threshold 0.441)

| Metric | Value |
| --- | --- |
| Recall | 71.1% |
| Precision | 55.0% |
| F1 | 0.620 |
| PR-AUC | 0.683 |

**Artifacts**

- Local train: `models/random_forest/model.joblib`
- Serving bundle: `serving/churn-rf/v1/`
- GCS (on deploy): `gs://<bucket>/models/churn-rf/v1/`

**Not in scope for v1**

- Engineered feature set
- XGBoost / LightGBM / LogReg deployment (local comparison only)

## v2 — (future)

Reserved for a second Registry version, e.g. retrain after split change,
new threshold policy, or a gated promotion experiment. When cutting v2:

1. Bump `SERVING_VERSION` in `src/champion.py` (or make it configurable).
2. Run `make train` → `make package` → update this file.
3. `make deploy` registers a new Vertex Model Registry version.
