# Phase 3 — Deploy Random Forest to Vertex AI

Package the **Random Forest champion** (not the CV winner) and deploy to Vertex Model Registry + Endpoint.

## Prerequisites

- Phase 2 complete: `make train` → `models/random_forest/model.joblib` + `metrics.json`
- `.env` filled from [`.env.example`](../.env.example)
- ADC: `gcloud auth application-default login`
- Billing budget alert set ([phase-0-setup.md](phase-0-setup.md))
- **Docker Desktop** installed and running (CPR builds the serving image locally, then pushes to Artifact Registry)

```bash
# After installing Docker Desktop:
docker --version
gcloud auth configure-docker us-west1-docker.pkg.dev
```

If `make deploy` fails with `FileNotFoundError: 'docker'`, install [Docker Desktop](https://www.docker.com/products/docker-desktop/), start it, then re-run deploy.

If registration fails with **Vertex AI Service Agent … does not have permission to access Artifact Registry**, re-run setup (grants IAM on the repo + bucket):

```bash
./scripts/setup_gcp.sh
make deploy REGISTER_ONLY=1
```

## Commands (in order)

### 1. Local package + smoke test (free)

```bash
make package        # copy RF -> serving/churn-rf/v1/
make package-test   # verify bundle matches pipeline scoring
make predict CUSTOMER_ID=7590-VHVEG   # compare local score
```

### 2. Dry run (no GCP changes)

```bash
make deploy DRY_RUN=1
```

### 3. Live deploy (costs while endpoint runs)

```bash
./scripts/setup_gcp.sh   # bucket, APIs, Artifact Registry repo
make deploy              # GCS upload + CPR image + Registry + Endpoint
make undeploy            # stop billing when done
```

Register without endpoint:

```bash
make deploy REGISTER_ONLY=1
```

## What gets deployed

```text
serving/churn-rf/
  CHANGELOG.md          # human release notes (git-tracked)
  v1/
    model.joblib          # full sklearn Pipeline (prep + RF)
    manifest.json         # release metadata (generated)
    threshold.json        # threshold + feature list + test metrics
    feature_schema.json   # 18 input columns
    predictor.py          # Vertex CPR ChurnPredictor
    requirements.txt      # serving dependencies
```

**Manifest vs changelog**

| Artifact | Purpose |
|---|---|
| `vN/manifest.json` | Machine-readable snapshot of this bundle: metrics, threshold, training config, git provenance. Generated by `make package`. |
| `CHANGELOG.md` | Human-readable history across releases (why v1, what changed in v2). Edit when cutting a new version. |
| Vertex Model Registry | GCP catalog; each `make deploy` creates a new Registry version with labels/description from manifest. |

Uploaded to `gs://{GCS_BUCKET}/models/churn-rf/v1/`.

## CPR predictor (simple explanation)

**Custom Prediction Routine (CPR)** is a Python class Vertex runs inside its prediction server:

1. **load** — read `model.joblib` + `threshold.json` from GCS
2. **preprocess** — validate 18 raw feature columns from JSON
3. **predict** — `pipe.predict_proba(...)`
4. **postprocess** — apply threshold → `{churn_probability, churn_flag}`

Vertex builds the Docker image from your `predictor.py` — you don't write a Flask app.

Registration sets `predictRoute=/predict` and `healthRoute=/health` on the CPR container (required for batch prediction jobs).
